### Tip 2: Keep Serial Monitor Open
When testing firmware, keep Arduino Serial Monitor open (115200 baud)
Provides debugging info and lets you send test commands

### Tip 3: Test API Manually
Before testing with firmware:
```bash
curl -X POST http://localhost:3000/api/device/light \
  -H "Content-Type: application/json" \
  -d '{"deviceId":"light","state":true}'
```

### Tip 4: Check Browser Console
Chrome (F12) shows frontend errors and logs
Very helpful for troubleshooting

### Tip 5: Read Troubleshooting
Each guide has a troubleshooting section
